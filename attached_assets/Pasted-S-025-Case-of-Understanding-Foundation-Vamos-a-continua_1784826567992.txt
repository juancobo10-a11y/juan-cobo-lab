Contexto conceptual

HELIOS evoluciona desde un motor metodológico hacia una plataforma de construcción de comprensión.

La nueva entidad raíz se denomina:

Case of Understanding / Caso de Comprensión

Un Caso de Comprensión representa el espacio dentro del cual se construye y evoluciona una explicación sobre un fenómeno complejo.

El motor metodológico actual debe quedar contenido dentro de un Caso de Comprensión.

Objetivo del sprint

Introducir la entidad UnderstandingCase como contexto raíz del análisis actual, preservando íntegramente el flujo ya construido.

El sprint debe permitir:

Crear un Caso de Comprensión.
Registrar su pregunta central.
Registrar su propósito, dominio, alcance y descripción.
Asociar el problema actual de HELIOS al caso.
Mostrar un resumen del caso antes de iniciar el flujo metodológico.
Incluir los datos del caso en snapshots, exportaciones e importaciones sin romper la compatibilidad existente.
Mantener operativos los proyectos exportados con esquema anterior mediante una migración segura, si resulta necesaria.
Modelo de dominio propuesto

Analiza y valida una interfaz semejante a esta, sin asumir que debe implementarse literalmente si existe una solución más coherente con los tipos actuales:

interface UnderstandingCase {
  id: string;
  name: string;
  understandingQuestion: string;
  purpose: string;
  domain: string;
  scope?: string;
  description?: string;
  status: "draft" | "active" | "in-review" | "archived";
  createdAt: string;
  updatedAt: string;
}

El campo actual problema: string no debe eliminarse.

Debe mantenerse como formulación específica del problema dentro del Caso de Comprensión.

La pregunta de comprensión y el problema son objetos conceptualmente distintos:

Pregunta de comprensión: aquello que se busca comprender.
Problema: formulación analítica del fenómeno que será estudiado.
