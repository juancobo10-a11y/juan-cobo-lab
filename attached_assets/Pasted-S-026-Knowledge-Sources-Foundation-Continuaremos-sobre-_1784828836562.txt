Objetivo

Introducir la entidad KnowledgeSource como una fuente de conocimiento asociada a un UnderstandingCase.

El usuario debe poder:

acceder a un espacio de Fuentes desde el contexto del caso;
registrar una fuente mediante texto pegado;
agregar metadatos básicos;
revisar el contenido registrado;
editar o eliminar una fuente antes de su procesamiento;
marcarla como lista para análisis;
incluir las fuentes en snapshots, exportaciones, importaciones, comparaciones y migraciones.
Modelo conceptual

Una fuente es un objeto que aporta información potencialmente relevante para construir comprensión.

Registrar una fuente no significa que su contenido sea verdadero, pertinente ni que constituya evidencia.

Debe preservarse esta distinción:

Fuente: origen del contenido.
Información: contenido incorporado.
Contribución: unidad cognitiva que podrá extraerse posteriormente.
Evidencia: información evaluada que respalda o contradice una hipótesis.

Experiencia de usuario

Añadir una pantalla o espacio denominado Fuentes dentro de HELIOS, accesible después de crear el caso.

La pantalla debe permitir:

visualizar las fuentes del caso;
crear una nueva fuente;
pegar texto;
completar metadatos;
guardar como borrador;
marcar como lista para análisis;
editar;
eliminar con confirmación;
consultar el contenido completo.

La interfaz debe mantener el sistema visual actual y preservar CaseSummaryBand.

No construyas todavía extracción de contribuciones. La acción “Analizar fuente” puede aparecer deshabilitada o marcada como próxima funcionalidad.
